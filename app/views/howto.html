How to

1. Create Movescount Apps

First you need to create two new apps in the App Designer on Movescount.com. Name the applications Int Duration and Int Target.

Make sure to set the publicity setting to Private for your apps. Movescount.com will not let you update an existing application if anyone else is using it - so by making your apps private you make sure no one else starts using your interval apps.

2. Add Own Variables

For the Int Target app you need to add the variables used by the app.

  • ACTUAL
  • TO
  • FROM
  • FORMATPACE
  • TARGET
  • TARGETSEC
  • TARGETMIN

3. Set Format

The output of an Movescount App can be formatted with 0, 1 or 2 decimals, or as time.

  • For the Int Target app, set the format to 2 decimals.
  • For the Int Duration set it to 0 decimals.

4. Create your interval session

Use the Interval Designer to create your interval session.

4.1 Step Type

Steps can be of different types. The type of interval is displayed as the prefix for the Int Duration app.

  • Other (othr)
  • Warm up (wu)
  • Interval (int)
  • Recovery (rec)
  • Rest (rest)
  • Cool down (cd)

4.2 Step Duration

Each step has a given duration. At the end of the duration (unless you use Lap as duration) the Int Duration app beeps to notify you to hit the lap button.

  • Distance (the app counts down a given distance)
  • Time (the app counts down a given time duration)
  • Calories (the app counts down number of burned calories)
  • Lap (the app counts up distance, untill the lap button is pressed)

4.3 Step Target

Each step has an optional target. The Int Target app will guide you to reach that target by displaying up, down or ok and the target values. The different types of targets are:

  • None (the app displays your current pace)
  • Pace (target a pace range, in min/km)
  • Cadence (target a cadence range - will display rpm for cycling activities, and rpm for running)
  • Speed (target a speed range, in kmh)
  • HR (target a heart rate range, in bpm)
  • Power (target a power range when cycling, in watt)

5. Copy your apps

5.1 Copy to clipboard

As you design your interval session, the source code for the Int Duration and Int Target apps are updated automatically. Once you are done desining your session, you need to copy the code into the Movescount App Designer.

Click the Select button in the interval designer to mark the code for the app, and hit Ctrl+C (Windows) or Cmd+C (Mac) to copy the code to your clipboard.

5.2 Enable advanced editing

To paste the app code into Movescount App Designer you need to click the Enable advanced editing link on Movescount. This will enable a text editor where you can paste the code by clicking Ctrl+V (Windows) or Cmd+V (Mac).

Click the Verify button on Movescount App Designer to verify that the app is correct, and save your app.

6. Add the apps to your watch

6.1 Create a new sports mode

You can either add the apps to an existing sports mode, or you can create a new one for your interval running.

6.2 Create a new screen

The Int Duration and Int Target apps are meant to be used together. I would recommend a three- or two-line screen, with the Int Duration app at the top, and the Int Target app at the bottom.

Uncheck the Log App checkboxes to prevent the watch from using memory to log the output from the Int Duration and Int Target apps. The output from these apps provide little value when analyzing a move.

7. Go for a run or ride

After synchronizing your watch's settings you are ready for a run or ride. Remember to hit the lap button once the Int Duration app reaches 0, and the alarm beeps. If you do not hit the lap button, the interval session will not progress to the next step.

8. Create a new interval set

When you are ready for a new workout, use the Interval Designer to design your session. Then simply replace the source code in the Movescount App Designer and synchronize your watch.
